- **缓解措施**: 
  - 详细设计状态转换图

### 2. 内存优化（点云暂存方案）

**问题**: 多路径多点位会缓存大量点云数据（200-450 MB）

**采用方案**: **流式处理 + PLY 文件存储**

#### 方案概述

不在内存中缓存点云数据，而是：
1. 每个点位采集完成后，立即保存为 PLY 文件
2. 内存中只缓存文件路径和位姿矩阵（约 1 KB/点位）
3. 蓝友算法从文件加载点云进行最终解算
4. 扫描完成后清理临时文件

#### 数据量对比

| 项目 | 全内存缓存 | 流式处理 |
|------|------------|----------|
| 内存占用 | 200-450 MB | 14 KB |
| 磁盘占用 | 0 | 200-450 MB |
| 访问速度 | 极快 | 快（SSD） |
| 可扩展性 | 差 | 优秀 |

#### 实现要点

**1. 点云保存**
```cpp
QString StateMachine::savePointCloudToPly(
    int pathId, 
    int pointIndex, 
    const PointCloudFrame& cloud) 
{
    // 生成文件名：path1_point3_20260519_143025.ply
    QString timestamp = QDateTime::currentDateTime().toString("yyyyMMdd_HHmmss");
    QString filename = QString("path%1_point%2_%3.ply")
        .arg(pathId).arg(pointIndex).arg(timestamp);
    
    QString dirPath = captureCacheRoot();  // 默认 <exe>/ScanTracking_CaptureCache/pointcloud/
    QDir().mkpath(dirPath);
    QString fullPath = dirPath + "/" + filename;
    
    // 保存为 PLY 格式（Binary 格式，节省空间）
    savePlyFile(fullPath, cloud);
    
    return fullPath;
}
```

**2. 轻量级结果缓存**
```cpp
struct ScanPointResult {
    int pathId;
    int pointIndex;
    
    // 只缓存文件路径，不缓存点云数据
    QString pointCloudPath;  // ← 关键：只存路径
    
    // 矩阵数据很小，可以缓存
    std::array<float, 16> calibrationMatrix;
    std::array<float, 16> stereoTrackingMatrix;
    
    qint64 timestampMs;
};

// 内存占用：约 1 KB/点位（14 点位 = 14 KB）
std::vector<ScanPointResult> m_pathResults;
```

**3. 采集完成回调**
```cpp
void StateMachine::onVisionBundleCaptureFinished(bundle) {
    // 1. 保存点云到临时文件
    QString plyPath = savePointCloudToPly(
        m_currentPathIndex, 
        m_currentPointIndex, 
        bundle.mechEyeResult.pointCloud);
    
    // 2. 只缓存文件路径和矩阵（轻量级）
    ScanPointResult result;
    result.pathId = m_currentPathIndex;
    result.pointIndex = m_currentPointIndex;
    result.pointCloudPath = plyPath;  // 文件路径
    result.calibrationMatrix = m_currentCalibrationMatrix;
    result.stereoTrackingMatrix = bundle.lbPoseResult.poseMatrix;
    result.timestampMs = QDateTime::currentMSecsSinceEpoch();
    
    m_pathResults.push_back(result);
    
    // 3. 立即释放内存中的点云数据
    bundle.mechEyeResult.pointCloud.clear();
    
    // 4. 继续下一个点位
    executeNextPoint();
}
```

**4. 蓝友算法调用**
```cpp
void StateMachine::onAllPathsCompleted() {
    // 准备输入数据（传递文件路径）
    MultiPathInput input;
    
    for (const auto& pathResult : m_pathResults) {
        PathData pathData;
        pathData.pathId = pathResult.pathId;
        
        // 传递文件路径，让蓝友算法自己加载
        for (const auto& pointResult : pathResult.pointResults) {
            pathData.pointCloudPaths.push_back(pointResult.pointCloudPath);
            pathData.calibrationMatrices.push_back(pointResult.calibrationMatrix);
            pathData.stereoTrackingMatrices.push_back(pointResult.stereoTrackingMatrix);
        }
        
        input.paths.push_back(pathData);
    }
    
    // 调用蓝友算法（从文件加载点云）
    auto result = m_lanyouAdapter->detectFromFiles(input);
    
    // 扫描完成后清理临时文件
    cleanupTempFiles();
}
```

**5. 临时文件清理**
```cpp
void StateMachine::cleanupTempFiles() {
    QString cacheDir = captureCacheRoot();
    
    // 删除本次扫描的所有 PLY 文件
    for (const auto& result : m_pathResults) {
        QFile::remove(result.pointCloudPath);
    }
    
    qInfo(LOG_FLOW) << "临时点云文件已清理";
}
```

#### 优势

- ✅ **内存占用极小**: 14 KB vs 200 MB（减少 99.99%）
- ✅ **支持超大规模扫描**: 理论上支持无限点位
- ✅ **便于调试**: PLY 文件可用于离线分析和回放
- ✅ **实现难度适中**: 不需要复杂的缓存管理

#### 注意事项

1. **磁盘空间**: 确保有足够的临时存储空间（建议预留 1 GB）
2. **SSD 优先**: 使用 SSD 可以显著提升文件读写速度
3. **异常处理**: 扫描失败时也要清理临时文件，避免磁盘泄漏
4. **调试模式**: 可选保留 PLY 文件（不自动清理），便于问题排查
