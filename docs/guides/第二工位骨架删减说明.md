# 第二工位 IPC 骨架删减说明

> **文档版本**：v1.2  
> **更新日期**：2026-06-16  
> **仓库**：`IPC_Station2`（第二工位专用，已从第一工位算法与流程剥离）  
> **读者**：IPC 开发 / 接手 AI Agent / HMI 联调

---

## 1. 目标与结论

本仓库现为 **第二工位专用 IPC 骨架**：保留相机采集、Modbus 握手、HMI TCP、9 个 PLC Handler 注册与占位实现；**移除**第一工位封头检测算法（Po_Kou / LB / LBN / Hole / Thickness）、`modules/tracking`、PCL/OpenCV 运行时依赖及第一工位配置/文档。

| 类别 | 状态 |
|------|------|
| 默认工位 | `stationId=2`，profile `station2_cylinder_semi.ini` |
| `Trig_ScanSegment` | **已实现**：scan_paths + 组合采集 + 段缓存 + `output/run_*` 落盘 |
| `Trig_Inspection` | **已实现（占位）** | 段缓存校验 + `publishInspectionResult`；算法待接入 |
| HMI TCP | **完整保留**（状态推送、`cmd.get_config`、`publishInspectionResult` 等） |
| 构建依赖 | Qt + Mech-Eye + Orbbec + Livox + HIK MVS（无 PCL/OpenCV/VTK） |

- **保留并接入**：`setInspectionResultPublisher`（HMI 检测推送）、`ScanSegmentCache`、`station2_inspection` 占位检测

### 2.3 配置与文档

- 删除 `config/station_profiles/station1_endcap.ini`、`config/scan_paths/station1_default.json`
- 三个重复的 `station2_*.json` 合并为 **`config/scan_paths/station2_placeholder.json`**
- 删除 `docs/station1/`、`docs/sequence_diagrams/station1/` 及根目录 `scan_paths_config.json`

## 3. 保留内容

### 3.1 相机与设备

- Mech-Eye、海康 CXP/Mono/C、`VisionPipelineService`（纯采集）
- Orbbec Gemini、Livox MID-360、Tfmini Plus
- Modbus 轮询 / 握手 / 超时 / 故障字

### 3.2 流程控制

9 个 Handler 均已注册（`modules/flow_control/src/handlers/`），业务在 Handler 内实现，通过 **`PlcTaskHost`**（`ctx.host`）调用 PLC/状态能力：

| 触发器 | 当前行为 |
|--------|----------|
| `Trig_ScanSegment` | Handler 发起组合采集；StateMachine 异步落盘；Res=1/5/6/7 |
| `Trig_Inspection` | Handler 调用 `evaluateInspectionForActiveTask` + `finishInspection` |
| `Trig_LoadGrasp` 等 | 各 Handler 占位实现（自检、位姿、编号等） |

架构详见 [flow_control架构说明.md](./flow_control架构说明.md)。

### 3.3 HMI（勿删）

- 模块：`modules/hmi_server/`
- 推送：`publishInspectionResult`、`publishInitialInspectionDisplay`、`status.*`、`event.scan.*` 等
- 协议占位（兼容旧显控）：
  - `cmd.set_bevel_recipe` → 返回「第二工位未实现」
  - `cmd.debug_trigger_inspection` → 从段缓存评估并 `publishInspectionResult`（**不写 PLC**）

第二工位检测实现后，在业务层调用：

```cpp
hmiTcpServer->publishInspectionResult(inspectionResult);
```

## 4. 默认配置

```ini
# config.ini
[Station]
stationId=2
profileIni=config/station_profiles/station2_cylinder_semi.ini
scanPathsConfigPath=config/scan_paths/station2_cylinder_semi.json
defaultWorkMode=MODE_CYLINDER_SEMI

[FlowControl]
pollIntervalMs=500
heartbeatIntervalMs=1000
simulatedProcessingMs=300
```

三种 profile 默认仍指向 `station2_placeholder.json`（封头/成品联调）；圆筒模式使用 **`station2_cylinder_semi.json`**（3 外部点）。详见 [扫描路径配置说明](./扫描路径配置说明.md)。

### 4.1 `station2_placeholder.json`

单段占位路径，供 Modbus 日志与 HMI `cmd.capture_bundle` 最小联调：

```json
{
  "version": "1.0",
  "scanPaths": [
    {
      "pathId": 1,
      "enabled": true,
      "totalPoints": 1,
      "points": [{ "pointIndex": 1, "needRotation": false }]
    }
  ]
}
```

### 4.2 `station2_cylinder_semi.json`

圆筒半成品外部扫描 3 点位（`segmentKind=external`）；内部伸缩杆段待硬件后追加。

## 5. 构建与运行

```bat
cmd /c tools\scan_tracking_dev.cmd configure-debug
cmd /c tools\scan_tracking_dev.cmd build-debug
```

- `CMakePresets.json` 已去掉 PCL/OpenCV/算法开关；Mech-Eye SDK 指向本仓库 `third_party`
