# 第二工位 IPC 骨架删减说明

> **文档版本**：v1.0  
> **更新日期**：2026-06-16  
> **仓库**：`IPC_Station2`（第二工位专用，已从第一工位算法与流程剥离）  
> **读者**：IPC 开发 / 接手 AI Agent / HMI 联调

---

## 1. 目标与结论

本仓库现为 **第二工位专用 IPC 骨架**：保留相机采集、Modbus 握手、HMI TCP、9 个 PLC Handler 注册与占位实现；**移除**第一工位封头检测算法（Po_Kou / LB / LBN / Hole / Thickness）、`modules/tracking`、PCL/OpenCV 运行时依赖及第一工位配置/文档。

| 类别 | 状态 |
|------|------|
| 默认工位 | `stationId=2`，profile `station2_cylinder_semi.ini` |
| `Trig_ScanSegment` / `Trig_Inspection` | Handler 已注册，触发后 **Res=8**（流程未实现） |
| HMI TCP | **完整保留**（状态推送、`cmd.get_config`、`publishInspectionResult` 等） |
| 构建依赖 | Qt + Mech-Eye + Orbbec + Livox + HIK MVS（无 PCL/OpenCV/VTK） |

## 2. 已删除内容

### 2.1 算法与模块

| 路径 | 说明 |
|------|------|
| `third_party/Po_Kou_Ce_Liang` | 坡口测量 |
| `third_party/LB`、`third_party/LBN` | 位姿跟踪 |
| `third_party/Hole`、`third_party/Thicknessmeasurement` | 开孔 / 壁厚 |
| `modules/tracking/` | 蓝友检测编排、`TrackingService` |
| `modules/mech_eye/point_cloud_processor.*` | PCL 位姿拼接 |
| `modules/vision/*_adapter.*` | 各算法适配层 |
| `cmake/OpenCvBundle.cmake`、`cmake/PclRuntime.cmake` | OpenCV/PCL 部署 |

### 2.2 状态机与流程

- `state_machine.cpp` 从约 3900 行精简至约 1100 行
- 移除：多路径点云缓存、LBN 标定、后台 PCL refinement、`executeBypassActiveTask`、`algorithmBypassEnabled`
- 移除：`cachedScanSegmentIndices`、`runDebugInspectionOnCachedSegments`、`setInspectionResultPublisher`

### 2.3 配置与文档

- 删除 `config/station_profiles/station1_endcap.ini`、`config/scan_paths/station1_default.json`
- 三个重复的 `station2_*.json` 合并为 **`config/scan_paths/station2_placeholder.json`**
- 删除 `docs/station1/`、`docs/sequence_diagrams/station1/` 及根目录 `scan_paths_config.json`

## 3. 保留内容

### 3.1 相机与设备

- Mech-Eye、海康 CXP/Mono/C、`VisionPipelineService`（纯采集）
- Orbbec Gemini、Livox MID-360、Tfmini Plus
- Modbus 轮询 / 握手 / 超时 / 故障字

### 3.2 流程控制

9 个 Handler 均已注册（`modules/flow_control/src/handlers/`）：

| 触发器 | 当前行为 |
|--------|----------|
| `Trig_ScanSegment` | 日志 + **Res=8** |
| `Trig_Inspection` | 日志 + **Res=8** |
| `Trig_LoadGrasp` 等 | 各占位实现（自检、位姿、编号等） |

### 3.3 HMI（勿删）

- 模块：`modules/hmi_server/`
- 推送：`publishInspectionResult`、`publishInitialInspectionDisplay`、`status.*`、`event.scan.*` 等
- 协议占位（兼容旧显控）：
  - `cmd.set_bevel_recipe` → 返回「第二工位未实现」
  - `cmd.debug_trigger_inspection` → 推送 Res=8 的 `event.inspection.finished` 并返回失败说明

第二工位检测实现后，在业务层调用：

```cpp
hmiTcpServer->publishInspectionResult(inspectionResult);
```

## 4. 默认配置

```ini
# config.ini
[Station]
stationId=2
profileIni=config/station_profiles/station2_cylinder_semi.ini
scanPathsConfigPath=config/scan_paths/station2_placeholder.json
defaultWorkMode=MODE_CYLINDER_SEMI

[FlowControl]
pollIntervalMs=500
heartbeatIntervalMs=1000
simulatedProcessingMs=300
```

三种 profile（`station2_endcap.ini` / `station2_cylinder_semi.ini` / `station2_semi_finished.ini`）均指向同一占位扫描路径 JSON；实现真实流程后再按模式拆分路径。

### 4.1 `station2_placeholder.json`

单段占位路径，供 Modbus 日志与 HMI `cmd.capture_bundle` 联调：

```json
{
  "version": "1.0",
  "scanPaths": [
    {
      "pathId": 1,
      "enabled": true,
      "totalPoints": 1,
      "points": [{ "pointIndex": 1, "needRotation": false }]
    }
  ]
}
```

## 5. 构建与运行

```bat
cmd /c tools\scan_tracking_dev.cmd configure-debug
cmd /c tools\scan_tracking_dev.cmd build-debug
cmd /c tools\scan_tracking_dev.cmd run-debug
```

- CMake 预设：`win-msvc2019-qtcore-ninja-debug` / `release`
- `CMakePresets.json` 已去掉 PCL/OpenCV/算法开关；Mech-Eye SDK 指向本仓库 `third_party`
- `tools/scan_tracking_dev.cmd`、`.vscode/launch.json` 的 PATH 已去掉 PCL/VTK/OpenNI2/OpenCV

## 6. 续接开发（第二步）

1. 阅读 [`工作流程-AI-Agent参考.md`](./工作流程-AI-Agent参考.md) 与对应时序图（`docs/sequence_diagrams/station2/`）
2. 在 Handler 中实现真实 `ScanSegment` / `Inspection`（替换 Res=8 占位）
3. 检测完成时调用 `HmiTcpServer::publishInspectionResult`
4. 按工位模式扩展 `scan_paths` 与 `station2_*.ini`（可保留 `station2_placeholder.json` 作联调最小集）

详见 [`双工位改造进度与续接指南.md`](./双工位改造进度与续接指南.md)。
