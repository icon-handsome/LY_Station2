## 1. 一句话现状

同一套 IPC 代码已通过 **配置切换工位**（`stationId=1|2` + `profileIni`），完成了双工位架构骨架和 Handler 分发；**扫描+检测主链路（第一工位）可联调**，但 **2D 有料、上/下料 3D、编号 OCR 等仍为占位**。你先去备硬件，回来从 **阶段 3** 接着做即可。

---

## 2. 总体策略（已定，勿改方向）

| 决策 | 内容 |
|------|------|
| 仓库策略 | **单仓库、单主线**，不用长期分支、不复制项目文件夹 |
| 工位区分 | `config.ini` `[Station]` + `config/station_profiles/*.ini` + `config/scan_paths/*.json` |
| 演进方式 | 第一工位作基点叠加第二工位能力，`stationId=1` 全程零回归 |
| PLC 通讯 | **Modbus 触发器框架保留**，阶段 3 起只填业务，不改寄存器地图（除非与 PLC 另开协议变更） |

## 4. 程序 vs 硬件：现在齐全吗？

**结论：主扫描链路较完整；阶段 3 需要的「上料/有料/下料/编号」硬件驱动有框架但业务未接。**

### 4.1 模块完成度一览

| 模块 / 触发器 | 代码状态 | 典型硬件 | 阶段 3 是否依赖 |
|---------------|----------|----------|-----------------|
| **Modbus PLC 通讯** | ✅ 完整 | PLC + 工控机网口 | 否（已有） |
| **Trig_ScanSegment** | ✅ 主流程已通 | Mech-Eye 3D + CXP 双目 | 封头/圆筒扫描都依赖 |
| **Trig_Inspection** | ✅ Po_Kou/Hole/Thickness | 同上 + 算法库 | 封头检测依赖 |
| **Trig_PoseCheck** | ⚠️ LB 已接，**扫段用 CXP 内存图；PoseCheck 可能仍读文件** | CXP 双目 | 封头模式依赖 |
| **LB / LBN 位姿** | ✅ 适配层在 | CXP + Mech-Eye | 扫描段依赖 |
| **Trig_StationMaterialCheck** | ❌ **占位 OK** | **2D 相机 + AI 有料** | **阶段 3 必须** |
| **Trig_LoadGrasp** | ❌ **占位**（环境变量/默认位姿） | **上料 3D 相机** | **阶段 3 必须** |

| **HikCxpCameraService** | ✅ 完整 | CH250 CXP 双目 | 封头段 LB |

### 4.2 阶段 3 之前建议备齐的硬件（封头模式 `MODE_END_CAP`）

按优先级：

| 优先级 | 硬件 | 用途 | 无硬件时阶段 3 能否开工 |
|--------|------|------|-------------------------|
| P0 | PLC + IPC 网络（Modbus） | 联调握手 | 可（已有） |
| P0 | **扫描工位 2D 相机**（或固定点 1 共用相机） | `Trig_StationMaterialCheck` AI 有料 | 可用离线图片 Mock，**现场必须** |
| P0 | **上料 3D 相机** | `Trig_LoadGrasp` 抓取点 | 可 Mock 点云，**现场必须** |
| P0 | **下料 3D 相机** | `Trig_UnloadCalc` 满料+下料点 | 可 Mock，**现场必须** |
| P1 | Mech-Eye + CXP（扫描主链路） | 已有第一工位配置 | 阶段 3 不改扫描可暂用现有 |
| P1 | CXP 双目（通孔位姿） | `Trig_PoseCheck` 改进 | 可先沿用现有 LB 路径 |
| P2 | 海康智能相机 C | `Trig_CodeRead` 编号 | 可后置 |
| P2 | 机械臂（PLC 控） | IPC 只输出位姿，不控臂 | 阶段 3 不必须 |

## 5. 后续阶段该干什么

### 阶段 3：封头模式真实业务补齐（下一步）

**目标**：第二工位 PDF 第 1 页（`MODE_END_CAP`）在 IPC 侧可端到端跑通（机械臂仍由 PLC 控）。

**软件任务**

| 触发器 | 从占位改为 | 建议新建/扩展 |
|--------|------------|---------------|
| `Trig_StationMaterialCheck` | 2D 采集 → AI 有/无料 → `Res_StationMaterialCheck` | `material_detection_adapter` |
| `Trig_LoadGrasp` | 上料 3D → 解算抓取点 → 40118–40128 | `load_grasp_3d_adapter` |
| `Trig_PoseCheck` | 固定点通孔扫描+位姿判定（接 CXP/3D 实时图） | 扩展 `PoseCheckHandler` |
| `Trig_UnloadCalc` | 下料 3D → 满料+下料点 → 40147–40157 | `unload_calc_3d_adapter` |
| `Trig_CodeRead` | 海康 C OCR → `kCodeValueAscii` | 接入 `HikCameraCController` |
| `Trig_Inspection` | 去掉现场强制 OK（可配置 `forceInspectionOk`） | `config.ini` 开关 |

**配置**

- 使用 `config/station_profiles/station2_endcap.ini`（或将第一工位 profile 复制改参数）
- `enableLoadGrasp/UnloadCalc/PoseCheck=true`

**不在本阶段做**

- 伸缩杆、吊装、碰撞、圆筒焊缝算法
- PLC 寄存器地址变更
- 彻底把 `execute*Task` 迁入 Handler 类（可顺手做，非必须）

**硬件前置**：见 §4.2 P0/P1。

**验收标准（摘要）**

- [ ] 有料/无料、通孔 NG、下料满料等分支 PLC 信号正确
- [ ] `stationId=1` 回归通过
- [ ] 各触发器不再依赖 `SCAN_TRACKING_*_POSE` 环境变量默认值（或有配置项替代）

### 阶段 4：第二工位扩展设备（并行安全 + 特有子系统）

**目标**：PDF 第 2–3 页的吊装、碰撞、辅助定位；与主流程**并行**运行。

**软件任务**

| 模块 | 职责 |
|------|------|
| `SafetyMonitorService` | 聚合 AI 人员、光栅、门锁 |
| `AIRobotClient` | 对接 AI 监控主机（文档 TODO §1.2） |
| `HoistAssistService` | 激光测距 + 2D 纵焊缝区域 → 三色灯 |
| `CollisionMonitorService` | 伸缩杆/吊装雷达 → 报警停机 |
| `TelescopicRodService` | 伸缩步进序列（20 cm） |
| `RollerBedService` | 滚轮 10° 步进 |

**配置生效**

- `enableTelescopicScan` / `enableHoistAssist` / `enableCollisionMonitor` 绑定模块创建与逻辑

**硬件前置**

- 激光测距 ×2、2D 相机（吊装辅助）
- 激光雷达 ×1 或 ×2（按圆筒/成品模式）
- 伸缩杆 + 末端扫描仪（圆筒模式）
- 滚轮架（PLC 或 RS485 协同）
- AI 监控工控机 + 通讯协议
