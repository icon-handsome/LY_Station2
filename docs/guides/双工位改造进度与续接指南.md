## 3. 已完成工作

### 3.1 架构骨架（原阶段 1～2，已适配第二工位专用仓）

- `StationProfile`、`config/station_profiles/station2_*.ini`
- 10 个 PLC 触发器 → `TaskHandlerRegistry` + Handler 分发（含 `Trig_TelescopicScan` 与 `Trig_ScanSegment` 分离）
- `enable*` 门控（LoadGrasp / UnloadCalc / PoseCheck 等）

### 3.2 第一工位剥离与精简（2026-06-16）

- 删除 `third_party` 算法库、`modules/tracking`、vision 适配层、`point_cloud_processor`
- 精简 `state_machine`、移除 `algorithmBypassEnabled` 与 PCL/OpenCV 构建链
- 合并扫描路径为 `station2_placeholder.json`
- HMI 保留并恢复 `cmd.set_bevel_recipe` / `cmd.debug_trigger_inspection` 协议占位

### 3.3 扫描主链路（2026-06-16）

- `Trig_ScanSegment`：`findScanPointByIndex` → `requestCaptureBundle` → 异步 Res=1
- `ScanSegmentCache` 内存缓存 + `output/run_{taskId}_{ts}/` 落盘（PLY/PNG/BMP）
- HMI：`event.scan.started` / `event.scan.finished`；`scanSegmentTotal` 取自 scan_paths 点位总数
- 配置：[`station2_cylinder_semi.json`](../../config/scan_paths/station2_cylinder_semi.json)（外部 3 点；内部伸缩杆段待硬件后扩展）

### 3.4 flow_control 架构重构（2026-06-16）

- `StateMachine` 拆分为多 cpp（Modbus / PLC IO / Scan / Inspection / helpers）
- 引入 **`PlcTaskHost`**：Handler 通过 `ctx.host` 调用 PLC 与 HMI 能力，无 `friend`
- 9 个 `Trig_*` 业务迁入 `handlers/*.cpp`（ScanSegment / Inspection 为真实或占位实现）
- 详见 [flow_control架构说明.md](./flow_control架构说明.md)

### 3.5 默认配置

```ini
# config.ini
[Station]
stationId=2
profileIni=config/station_profiles/station2_cylinder_semi.ini
scanPathsConfigPath=config/scan_paths/station2_cylinder_semi.json
defaultWorkMode=MODE_CYLINDER_SEMI
```

切换封头/成品模式：改用 `station2_endcap.ini` 或 `station2_semi_finished.ini`（扫描路径暂仍指向 placeholder，待按模式拆分 JSON）。
