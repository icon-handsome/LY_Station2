> **IPC 实现说明（2026-05-25）**：下图「扫描 5～6 段」为 PLC/工艺视角；IPC 每段 `Trig_ScanSegment` 落盘至 `ScanTracking_CaptureCache`，**综合检测**时蓝友仅融合 `[Tracking]` 配置的 **3 段**点云（非全部段）。详见 [`多点位扫描与位姿跟踪完整流程.md`](./多点位扫描与位姿跟踪完整流程.md) v1.3。

```mermaid
sequenceDiagram
    autonumber
    participant O as 操作员
    participant P as PLC
    participant M as 转台/机械机构
    participant I as IPC
    participant MES as MES

    P->>P: 确认安全正常、任务有效、扫描位可用
    P->>M: 放料到扫描位

    P->>I: 请求有无料检测
    I->>I: 判断有无料并解算初始位姿/抓取点
    alt 放行通过
        I->>P: 返回有料信号和初始位姿坐标
    else 无料/位姿异常
        I->>P: 返回无料/异常
        P->>O: 报警并等待人工处理
    end

    loop 扫描5~6段
        P->>M: 驱动到本段目标位
        P->>I: 触发本段扫描
        I->>I: 执行本段扫描
        alt 本段成功
            I->>P: 返回本段完成
        else 本段失败
            I->>P: 返回本段失败
            P->>O: 报警并中止本件
        end
    end

    I->>I: 拼接并整体解算
    I->>P: 返回扫描完成和结果(OK/NG)

    P->>M: 机械臂到下料位
    P->>I: 请求下料坐标
    I->>I: 解算下料坐标并判断下料区是否满料
    alt 下料区未满
        I->>P: 返回下料坐标
        P->>M: 按坐标执行下料
        P->>MES: 上报检测结果
    else 下料区满料
        I->>P: 返回满料信号
        P->>O: 报警并等待清料
    end
```
