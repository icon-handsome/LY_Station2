# 第二工位扫描路径配置说明

> **文档版本**：v1.0  
> **更新日期**：2026-06-16  
> **配置文件目录**：`config/scan_paths/`  
> **加载入口**：`config.ini [Station] scanPathsConfigPath` 或 `station_profiles/*.ini`

---

## 1. 用途

`scan_paths` JSON 定义 PLC `Trig_ScanSegment` 触发的**全局段号（pointIndex）**与采集策略。IPC 按段号查配置后调用 `VisionPipelineService::requestCaptureBundle` 执行 Mech-Eye + 海康 CXP 组合采集。

| 模式 | 默认配置文件 |
|------|----------------|
| 圆筒半成品 `MODE_CYLINDER_SEMI` | [`station2_cylinder_semi.json`](../../config/scan_paths/station2_cylinder_semi.json) |
| 联调最小集 | [`station2_placeholder.json`](../../config/scan_paths/station2_placeholder.json) |
| 封头 / 成品 | 暂共用 placeholder，后续按模式拆分 |

---

## 2. JSON 结构

```json
{
  "version": "1.0",
  "lastModified": "2026-06-16",
  "scanPaths": [
    {
      "pathId": 1,
      "enabled": true,
      "segmentKind": "external",
      "description": "扫描机械臂外部多点位",
      "totalPoints": 3,
      "points": [
        { "pointIndex": 1, "needRotation": false },
        { "pointIndex": 2, "needRotation": false },
        { "pointIndex": 3, "needRotation": true }
      ]
    }
  ]
}
```

### 2.1 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `pathId` | 是 | 路径 ID，全局唯一 |
| `enabled` | 否 | 默认 `true`；`false` 时该路径下点位不参与扫描 |
| `segmentKind` | 否 | 段类型，默认 `external`；后续可扩展 `internal_longitudinal` / `internal_ring`（伸缩杆/滚轮） |
| `description` | 否 | 说明文字，仅日志/文档用 |
| `totalPoints` | 是 | 本路径点位数量，须与 `points` 数组长度一致 |
| `points[].pointIndex` | 是 | **全局段号**，与 PLC 寄存器 `ScanSegmentIndex` 一致 |
| `points[].needRotation` | 否 | `true` → Mech-Eye `Capture2DAnd3D`；`false` → `Capture3DOnly` |

### 2.2 校验规则（`ConfigManager::validateScanPathsConfig`）

- 路径 `pathId` 不重复
- `totalPoints` 与 `points` 数量一致
- 已启用路径中 **全局段号 `pointIndex` 不重复**且大于 0
- 多路径时，各路径可使用不同段号区间（如外部 1–3、内部 4–N）

### 2.3 `scanSegmentTotal`

HMI `status.plc.scanSegmentTotal` 与 StateMachine 任务上下文取 **`enabledScanPointCount()`**（所有已启用路径的点位总数）。若 JSON 未加载或为空，回退 `config.ini [Tracking] scanSegmentTotal`。

## 3. 运行时行为（已实现）

```text
PLC Trig_ScanSegment + segmentIndex
  → findScanPointByIndex(segmentIndex)
  → requestCaptureBundle（按 needRotation 选 2D+3D / 3D）
  → 成功：内存缓存 + 落盘 output/run_{taskId}_{timestamp}/
  → Res=1，写 kScanSegmentDoneIndex，推送 event.scan.*
```

### 3.1 落盘目录

单次 PLC `taskId` 共用一个 run 目录（与 exe 同级的 `output/` 下）：

```text
output/run_{taskId}_{yyyyMMdd_HHmmss_zzz}/
  mech_3d/segment_{N}_task{T}_{ts}.ply
  mech_2d/segment_{N}_task{T}_{ts}.png   （needRotation 且 2D 有效时）
  hik_mono/camera_a/segment_{N}_task{T}_{ts}.bmp
  hik_mono/camera_b/segment_{N}_task{T}_{ts}.bmp
```

### 3.2 段缓存

- 每段采集成功后写入 `ScanSegmentCache`（内存），供后续 `Trig_Inspection` 读取（**待实现**）
- `Trig_ResultReset` 调用 `resetScanSegmentCache()` 清空

### 3.3 Res 码（ScanSegment）

| Res | 含义 |
|-----|------|
| 1 | 采集成功 |
| 5 | 段号不在 scan_paths 中 |
| 6 | 任务超时 |
| 7 | 视觉采集失败 / 流水线不可用 |

---

### 3.4 Res 码（Inspection）

| Res | 含义 |
|-----|------|
| 1 | OK（当前占位：段缓存校验通过） |
| 2 | NG（段 bundle 无效） |
| 3 | 需人工复检（无缓存 / taskId 不一致 / 段数不足） |
| 6 | 任务超时 |

流程：

```text
PLC Trig_Inspection
  → evaluateStation2Inspection(ScanSegmentCache)
  → writeInspectionResult(NG_Reason*) + Res
  → publishInspectionResult → event.inspection.finished
```

HMI 调试：`cmd.debug_trigger_inspection` 仅评估缓存并推送，**不写 PLC**。
